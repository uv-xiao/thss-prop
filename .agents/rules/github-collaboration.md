# GitHub Collaboration Rules

- Use GitHub work to preserve reviewability of report changes, not to create process overhead.
- Keep report edits, bibliography edits, terminology edits, and harness edits logically grouped when possible.
- Before creating a PR, run `make pdf` unless the change is documentation-only and cannot affect the report build.
- PR bodies must include writing-scope summary, citation impact, terminology impact, verification commands, and residual risks.
- Reviews must prioritize argument gaps, unsupported claims, mistranslations, inconsistent terminology, broken Typst output, and oversized or underspecified sections.
- Do not force-push without explicit user approval.
- Do not rewrite existing user-authored report text merely for style if it changes meaning or removes intended emphasis.
- Do not merge or close GitHub work without confirming the desired branch and repository state.
