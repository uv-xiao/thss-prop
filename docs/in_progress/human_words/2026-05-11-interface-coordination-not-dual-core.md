# User Feedback: Interface Coordination, Not Dual Core

Source wording:

> 一个问题是，是否目标能力只放在敏捷芯片设计体系上，因为这样做会弱化编译支持的分量

> 双核心这个表述不好，”协同“更好，因为双核心有种分离、并列的感觉，不够整体

Initial interpretation:

- The report should not frame the target capability only as a "敏捷芯片设计体系", because that makes compilation support appear secondary.
- The report should also avoid "双核心" for architecture and compilation. Although it gives compilation equal weight, it implies two separated centers.
- Preferred framing: architecture and compilation should be written as a coordinated relation inside one hardware-software co-design system. Compilation and agile chip design mutually define, constrain, and validate each other.
